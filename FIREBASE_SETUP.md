# 端末間データ同期（Firebase）のセットアップ手順

このアプリに「他の端末とデータを共有する」機能を追加しました。
使う前に、**あなた専用のFirebaseプロジェクト**を1つ作る必要があります（無料・5分程度）。

---

## ① Firebaseプロジェクトを作る

1. [https://console.firebase.google.com/](https://console.firebase.google.com/) を開き、Googleアカウントでログイン
2. 「プロジェクトを作成」をクリック
3. プロジェクト名を入力（例：`jikanwari`）→「続行」
4. Googleアナリティクスは**オフ**のままでOK →「プロジェクトを作成」
5. 作成が終わったら「続行」

---

## ② Firestoreデータベースを有効にする

1. 左側メニューから「構築」→「Firestore Database」を選択
2. 「データベースの作成」をクリック
3. ロケーションはそのまま（またはお好きな場所）→「次へ」
4. **「テストモードで開始する」を選択** →「有効にする」

   > テストモードは30日間だけ誰でも読み書きできる設定です。期限が切れると使えなくなるので、
   > 長く使いたい場合は下の「④ セキュリティルールの設定（推奨）」も行ってください。

---

## ③ ウェブアプリを登録して設定値を取得する

1. プロジェクトのトップ画面に戻り、「`</>`」（ウェブ）アイコンをクリック
2. アプリのニックネームを入力（例：`timetable-web`）→「アプリを登録」
3. 表示された `firebaseConfig` の中身をコピーする（下記のような形です）

```js
const firebaseConfig = {
  apiKey: "AIza...",
  authDomain: "jikanwari-xxxx.firebaseapp.com",
  projectId: "jikanwari-xxxx",
  storageBucket: "jikanwari-xxxx.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef123456"
};
```

4. このプロジェクトの `firebase-config.js` を開き、`"ここに◯◯を貼り付け"` の部分を、
   コピーした値にひとつずつ書き換えて保存してください。

---

## ④ セキュリティルールの設定（推奨・任意）

そのままでも30日間は動きますが、期限を気にせず使いたい場合はルールを変更してください。

1. Firebaseコンソール →「Firestore Database」→「ルール」タブ
2. 以下の内容に書き換えて「公開」

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /timetables/{code} {
      allow read, write: if true;
    }
  }
}
```

> ⚠️ この設定は「同期コードを知っていれば誰でも読み書きできる」状態です。
> 個人利用なら問題ありませんが、コードを他人に教えないようにしてください。

---

## ⑤ 使い方

1. アプリを開き、右上の「同期」ボタンをタップ
2. 1台目の端末：「OK」を選んで新しい同期コード（6桁）を作成
3. 表示されたコードをメモ or 家族・自分の別端末に伝える
4. 2台目の端末：同じく「同期」ボタン →「キャンセル」→ コードを入力

これで、どちらかの端末で予定や時間割を変更すると、もう片方にも自動で反映されます。

---

## 注意点

- 通知のON/OFFは端末ごとの設定なので同期されません
- 同期コードを知っている人は誰でもそのデータを読み書きできます。コードは家族や自分だけで共有してください
- インターネット接続がない状態でも今まで通りアプリ自体は使えます（ローカル保存が優先されます）。オンラインに戻ると自動で同期されます
